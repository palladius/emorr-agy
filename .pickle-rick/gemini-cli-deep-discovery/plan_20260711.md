# Plan: Deeper Gemini CLI Session Discovery
**Date**: 2026-07-11
**Ticket**: gemini-cli-deep-discovery

## Approach

Build a `projectHash → realPath` reverse-lookup table by scanning known parent directories, then use it to resolve ALL Gemini CLI `~/.gemini/tmp/` folders.

## Implementation Steps

- [ ] **Step 1**: Add `buildPathHashTable()` method to `ClassificationEngine`
  - Scan `~/git/*`, `~/*`, `~/obsidian/*`, `~/obsidian-pbt/*` directories
  - For each dir, compute `sha256(absolutePath)` → store in `map[string]string` (hash→path)
  - Cache result on the engine so it's computed once per Classify() call

- [ ] **Step 2**: Add `resolveGeminiFolder()` method
  - Input: folderName (the `~/.gemini/tmp/<name>` basename)
  - If len(folderName) == 64 (SHA256): direct lookup in hash table
  - If folderName has `chats/*.jsonl`: read first line → get `projectHash` → lookup in hash table
  - Fallback: try existing candidates (`~/git/<name>`, `~/<name>`)
  - Return resolved path or ""

- [ ] **Step 3**: Replace the inline folder resolution in section 2d (lines 460-471) with call to `resolveGeminiFolder()`

- [ ] **Step 4**: Test — run `sessions list` and verify previously unresolved sessions now show correct folders

- [ ] **Step 5**: Commit and push

## Risks
- **Performance**: Scanning many dirs for hash table. Mitigated: only top-level entries in a few parent dirs, cached.
- **Hash collisions**: SHA256 = zero risk in practice.
