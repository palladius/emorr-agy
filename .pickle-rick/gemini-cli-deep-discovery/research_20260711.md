## Key Discovery: SHA256 Hashing

Gemini CLI uses two folder naming schemes in `~/.gemini/tmp/`:
1. **Named dirs**: basename of the last path component (e.g. `gic25` for `~/git/gic25`)
2. **SHA256 hash dirs**: `sha256(absolute_path)` of the working directory (e.g. `ad18ef26...` for `/home/riccardo/git/gic25`)

## Key Discovery: `chats/*.jsonl` → `projectHash`

Each chat session file (`chats/session-*.jsonl`) has a first-line JSON with `projectHash` = `sha256(real_working_dir)`. This is the authoritative mapping.

## Resolution Strategy

1. **Named dirs**: Expand candidate parents to `~/git/`, `~/`, `~/obsidian/`, `~/obsidian-pbt/`, `~/.hermes-freevola/`
2. **Hash dirs**: Already IS the hash. Just need to reverse-lookup.
3. **Universal**: Read `chats/*.jsonl` first line → get `projectHash` → reverse-lookup via precomputed table.

The reverse-lookup table: for each known parent dir (`~/git/*`, `~/*`), compute `sha256(path)` and store in a map. Then for any `projectHash`, look up the real path.
