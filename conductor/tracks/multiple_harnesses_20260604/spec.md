# Specification: Support Multiple Harnesses (Gemini CLI and Claude Code)

## Overview
Currently, `emorr-agy` only monitors active conversations generated by the Antigravity CLI (`agy`) by checking file descriptors for `.db` SQLite databases in `~/.gemini/antigravity-cli/conversations/`. 
This track expands `emorr-agy` to auto-discover and monitor sessions from other harnesses, specifically:
1. **Gemini CLI (`gemini`)**: Running sessions stored as JSON/JSONL files in `~/.gemini/tmp/` (under workspace directory structures).
2. **Claude Code (`claude`)**: Running sessions (we will auto-detect active Claude processes).

The CLI will dynamically monitor and display sessions for all active harnesses on the machine.

## Functional Requirements
- **Auto-Discovery of Harnesses**: Check which binaries (`agy`, `gemini`, `claude`) are installed on the system using path lookups (`which` or similar).
- **Session Scan per Harness**:
  - **Antigravity CLI (`agy`)**: Scan `~/.gemini/antigravity-cli/conversations/` (existing SQLite DB scan logic).
  - **Gemini CLI (`gemini`)**: Scan `~/.gemini/tmp/*/chats/` for active/recent JSON/JSONL sessions.
  - **Claude Code (`claude`)**: Scan for running `claude` processes and their history.
- **Unified Monitor Output**: Merge and display active sessions/threads from all detected harnesses in the `monitor` and `status` commands. Use distinct emojis/tags to represent each harness (e.g. ♊ for Gemini CLI, 🟢 for Ag, and 🐜 for Claude Code).
- **Process Inference**: Infer if a session is active by checking if there is a running process of that harness associated with the session directory/file.

## Acceptance Criteria
1. Running `emorr-agy monitor` shows active sessions from both `agy` and `gemini` (if active).
2. The display indicates the harness name/type of each active thread.
3. The `status` command lists the counts of active threads grouped by harness.
